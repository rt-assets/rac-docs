v1.0.2 · in development

since Apr 21 In progress
Latest · Apr 30

Long cables and grab-and-walk get a serious overhaul

WrappingCable now scales particle count to cable length automatically (new TargetSegmentLength + MaxParticlesPerCable), wrap formation no longer pauses while you're holding the cable, and contact friction releases once the chain is over-stretched so cables flow past obstacles instead of getting glued. Substep gravity is properly compensated, visibility refreshes continuously on long droopy cables, and overlap capsules auto-grow with length so the whole rope stays grabbable. The portable subset of the same fixes lands on the regular cable too.

v1.0.2 · WrappingCable + Verlet cable
Feature · Apr 26 Spline Cable: place cable-shaped curves in the level New ARACSplineCable actor. Edit a spline component, the cable follows. No public API breaks.
Feature · Apr 26 Cable Point: drive endpoint physics + inextensible toggles Two new flags on URACCablePointComponent so multi-cable rigs (chandeliers, swings, anything with more than one rope on a body) actually behave.
Fix · Apr 26 Inextensible multi-cable: chains share the load Hanging a body from multiple bPreventStretching cables used to let the body drift because each cable overwrote the others. Now they apply additive impulses with Baumgarte position correction. All cables contribute every frame.
Fix · Apr 26 Attach offsets follow actor rotation and scale Start/End offsets are now actor-local-space, so rotating an attachment rotates its offsets too. Symmetric chandelier rigs no longer tip over.
Polish · Apr 26 Cleaner detail panel Redundant Cable Actor categories hidden in the editor. Show Debug Lines toggle on Cable Point now actually hides the lines.
Polish · Apr 26 Editor preview matches PIE In-editor preview uses a mock Verlet integrator so the shape you see in the viewport matches what you get in play.
Fix · Apr 26 Wrap points no longer trigger on flat floors Constrain at Wrap Points now ignores the floor so cables draped across surfaces stop snapping at random.

v1.0.1 · live on Fab

Apr 21 Shipping now
Feature Swing follows moving platforms Grapple onto a moving train and you swing with it. Three new toggles control whether you follow, inherit platform velocity on release, or attach to actor origin Pull-style.
Feature Pull grabs at the hit point New bAttachToHitPoint on Pull. Rope attaches where the trace landed and applies torque, so corner-grabs tumble naturally.
Fix Swing auto-releases if the target is destroyed No more frozen ropes hanging in mid-air after the thing you grabbed gets blown up.
Demo Cable Web station rebalanced Updated meshes, better cable spacing, intro tutorial prompt fixed.
Polish Appearance category re-org Detail panel ordering and grouping for cable visuals is now consistent with how you actually edit them.

v1.0.0 · launched

Apr 7 Released
Genesis

RAC went public on Fab.

14 runtime classes. 7 demo stations. UE 5.4 / 5.5 / 5.6 / 5.7. The whole point: ready-made cable gameplay (Swing, Climb, Pull, Tow, Zipline, Tether, Interaction, Pulley) plus a topology-aware wrapping cable, a custom Niagara DI for VFX-driven cables, and multi-tier LOD. Built so you can drop a marker, get a cable, wire up gameplay in a Blueprint without touching C++.
